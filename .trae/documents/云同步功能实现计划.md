# 云同步功能实现计划

## 一、架构设计

### 1.1 云存储方案选择

**方案A：自建后端服务（推荐）**

* 使用 Node.js + Express/Fastify 构建简单的 RESTful API

* 使用 MySQL存储用户数据

* 优点：完全可控，数据安全，成本低

* 缺点：需要维护服务器

**方案B：第三方云服务集成**

* 集成 Google Drive API、Dropbox API 或 iCloud

* 优点：无需自建服务器，跨设备同步方便

* 缺点：依赖第三方服务，可能存在限制

**建议**：初期采用方案A，后续可扩展方案B

### 1.2 数据结构设计

需要同步的数据类型：

* 用户配置（AppConfig）- 主题、字体等

* SSH 连接配置（SSHConnection）- 需要加密存储

* 连接组（ConnectionGroup）

* 密钥管理（可选）

### 1.3 安全设计

* 使用 AES-256 加密敏感数据（SSH密码、私钥）

* HTTPS/TLS 传输加密

* JWT Token 认证

* 端到端加密可选

### 2.2 API 接口设计

```
POST   /api/auth/register  - 用户注册
POST   /api/auth/login     - 用户登录
POST   /api/auth/logout    - 用户登出

GET    /api/config         - 获取云端配置
POST   /api/config         - 上传配置
PUT    /api/config         - 更新配置
DELETE /api/config         - 删除配置

GET    /api/connections    - 获取连接列表
POST   /api/connections    - 上传连接列表
PUT    /api/connections    - 更新连接列表

GET    /api/sync/status    - 获取同步状态
POST   /api/sync/push      - 推送本地数据到云端
POST   /api/sync/pull      - 从云端拉取数据
```

### 2.3 冲突解决策略

* **时间戳策略**：比较最后更新时间，保留最新的

* **合并策略**：对于可合并的数据（如主题配置），进行合并

* **用户选择**：对于冲突严重的场景，让用户选择保留哪一端

## 五、技术要点

### 5.1 数据加密

```typescript
// 使用 crypto-js 加密敏感数据
import CryptoJS from 'crypto-js';

const encrypt = (data: string, secret: string) => {
  return CryptoJS.AES.encrypt(data, secret).toString();
};

const decrypt = (encrypted: string, secret: string) => {
  const bytes = CryptoJS.AES.decrypt(encrypted, secret);
  return bytes.toString(CryptoJS.enc.Utf8);
};
```

### 5.2 冲突检测

```typescript
const hasConflict = (local: any, remote: any) => {
  return local.lastModified !== remote.lastModified && 
         local.lastModified > remote.lastModified;
};
```
